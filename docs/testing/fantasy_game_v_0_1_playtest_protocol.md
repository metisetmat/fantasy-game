# Fantasy Game — V0.1 Playtest Protocol

# Objectif du protocole

Ce protocole sert à évaluer la première version jouable du moteur de simulation.

Le but n’est PAS de vérifier :
- la qualité graphique ;
- l’interface ;
- le réalisme visuel.

Le but est de vérifier :
- la cohérence tactique ;
- la lisibilité des matchs ;
- l’émergence des styles ;
- l’équilibrage systémique ;
- la compréhension des rapports ;
- le potentiel émotionnel du jeu.

---

# Philosophie générale

Le protocole repose sur une idée centrale :

> Un bon moteur de simulation doit produire des matchs qu’un humain peut lire, interpréter et comprendre.

Le moteur doit :
- raconter une histoire ;
- produire des dynamiques crédibles ;
- générer des styles reconnaissables ;
- permettre l’analyse tactique.

---

# Structure générale du protocole

Le protocole se déroule en 3 niveaux.

| Niveau | Objectif |
|---|---|
| Niveau 1 | Validation systémique brute |
| Niveau 2 | Validation humaine des rapports |
| Niveau 3 | Validation cognitive multi-profils |

---

# NIVEAU 1 — Validation systémique brute

# Objectif

Vérifier que le moteur produit :
- des matchs variés ;
- des styles cohérents ;
- des statistiques plausibles ;
- des interactions crédibles.

---

# Méthode

Le moteur simule automatiquement :
- 500 à 1000 matchs.

Sans intervention humaine.

---

# Ce qui est observé

## 1. Diversité des matchs

Chercher :
- variété des scores ;
- variété des tempos ;
- variété des turnovers ;
- variété des dynamiques.

---

## 2. Dominance tactique excessive

Détecter si :
- une stratégie domine systématiquement ;
- une combinaison d’attributs casse le jeu ;
- un rôle devient obligatoire.

---

## 3. Comportements extrêmes

Chercher :
- scores absurdes ;
- trop peu d’occasions ;
- trop de chaos ;
- trop peu de turnovers ;
- trop de turnovers.

---

## 4. Impact de la fatigue

Vérifier :
- baisse qualité technique ;
- baisse coordination ;
- baisse pressing ;
- augmentation erreurs.

---

# Critères de succès Niveau 1

## Succès si :

- plusieurs styles sont viables ;
- aucun style ne domine massivement ;
- les matchs racontent des dynamiques différentes ;
- les styles produisent des conséquences cohérentes ;
- la fatigue agit comme régulateur systémique.

---

# NIVEAU 2 — Validation humaine

# Objectif

Tester si un coach humain peut :
- lire ;
- comprendre ;
- interpréter ;
- analyser les matchs.

---

# Méthode

Le testeur humain reçoit :
- 3 à 5 rapports de match.

MAIS :
- sans connaître les attributs ;
- sans connaître les consignes ;
- sans connaître les profils exacts.

Le moteur doit “parler” à travers les rapports.

---

# Informations fournies au testeur

## 1. Highlights narratifs

Exemples :
- récupérations hautes ;
- changements d’aile ;
- attaques verticales ;
- séquences longues ;
- situations de chaos.

---

## 2. Rapport collectif

Exemples :
- domination territoriale ;
- pressing ;
- densité ;
- transitions ;
- fatigue ;
- utilisation des couloirs.

---

## 3. Statistiques collectives

Exemples :
- possessions ;
- turnovers ;
- récupérations hautes ;
- zones d’attaque ;
- tirs ;
- drops ;
- pertes sous pression.

---

## 4. Statistiques individuelles

Exemples :
- implication ;
- passes ;
- percées ;
- récupérations ;
- impacts ;
- erreurs ;
- fatigue finale.

---

## 5. Rapport du coach adjoint

Analyse simplifiée et interprétation tactique.

---

# Questions posées au testeur

## Objectif principal

Voir si le testeur parvient à identifier :
- les styles ;
- les points forts ;
- les points faibles ;
- les profils de joueurs ;
- les causes des bascules du match.

---

# Exemples de questions

## Styles

- Quelle équipe semblait contrôler le tempo ?
- Quelle équipe cherchait la verticalité ?
- Quelle équipe utilisait le plus le jeu au pied ?
- Quelle équipe semblait jouer dans la transition ?

---

## Structure

- Quelle équipe semblait la plus compacte ?
- Quelle équipe semblait mieux organisée ?
- Quelle équipe exploitait le mieux les couloirs ?

---

## Fatigue

- Quelle équipe semblait fatiguer le plus vite ?
- À quel moment le match a-t-il basculé physiquement ?
- Quels signes de fatigue étaient visibles ?

---

## Joueurs

- Quel joueur semblait être le moteur du jeu ?
- Quel joueur semblait le plus dangereux ?
- Quel joueur semblait relier les lignes ?
- Quel joueur semblait protéger la structure ?

---

## Transitions

- Quelle équipe semblait la plus dangereuse après récupération ?
- Quelle équipe survivait le mieux après perte de balle ?

---

# Critères de succès Niveau 2

## Succès si :

Le testeur humain parvient à :
- reconnaître les styles ;
- expliquer les dynamiques ;
- identifier les profils ;
- comprendre les bascules du match ;
- proposer des ajustements tactiques.

---

# Échec si :

Le testeur :
- ne comprend pas ce qui se passe ;
- voit les matchs comme aléatoires ;
- ne distingue pas les équipes ;
- ne détecte aucun style.

---

# NIVEAU 3 — Validation cognitive multi-profils

# Objectif

Évaluer si le moteur “parle” à différents types de joueurs.

---

# Principe

Des personas analytiques virtuels interprètent les rapports.

Chaque persona possède :
- des priorités ;
- une sensibilité différente ;
- une façon particulière de lire un match.

---

# Persona 1 — Coach Football Tactique

## Sensible à

- structure ;
- pressing ;
- espaces ;
- transitions ;
- occupation terrain.

---

## Cherche

- cohérence ;
- domination spatiale ;
- contrôle du tempo.

---

## Risques détectés

- espaces irréalistes ;
- pressing incohérent ;
- structure illisible.

---

# Persona 2 — Fantasy Optimizer

## Sensible à

- statistiques ;
- attributs ;
- builds ;
- abus système.

---

## Cherche

- méta ;
- combinaison dominante ;
- optimisation maximale.

---

## Risques détectés

- attribut trop fort ;
- rôle obligatoire ;
- stratégie cassée.

---

# Persona 3 — Coach Occasionnel

## Sensible à

- narration ;
- émotions ;
- stars ;
- rythme du match.

---

## Cherche

- matchs mémorables ;
- lisibilité ;
- moments forts.

---

## Risques détectés

- matchs répétitifs ;
- rapports trop techniques ;
- manque d’émotion.

---

# Persona 4 — Simulation Hardcore

## Sensible à

- cohérence systémique ;
- fatigue ;
- rôles ;
- émergence ;
- profondeur.

---

## Cherche

- subtilité ;
- variété ;
- cohérence tactique.

---

## Risques détectés

- comportements scripts ;
- fatigue artificielle ;
- manque d’émergence.

---

# Persona 5 — Chaos Gamer

## Sensible à

- transitions ;
- prises de risque ;
- variance ;
- agressivité.

---

## Cherche

- spectacle ;
- momentum ;
- chaos contrôlé.

---

## Risques détectés

- matchs trop fermés ;
- manque de renversements ;
- rythme trop lent.

---

# Exploitation des personas

Chaque persona produit :
- des observations ;
- des frustrations ;
- des hypothèses ;
- des alertes équilibre.

---

# Objectif final du protocole

Le moteur V0.1 sera considéré comme réussi si :

- les styles sont reconnaissables ;
- les matchs racontent des histoires ;
- la fatigue régule naturellement les excès ;
- plusieurs philosophies sont viables ;
- les coachs veulent expérimenter ;
- les rapports permettent une vraie lecture tactique.

---

# Vision long terme

À terme, ce protocole doit devenir :
- un outil d’équilibrage permanent ;
- un outil de détection des métas ;
- un outil de validation des nouvelles mécaniques.

Le moteur doit toujours rester :
- lisible ;
- émergent ;
- tactique ;
- crédible ;
- émotionnellement engageant.
